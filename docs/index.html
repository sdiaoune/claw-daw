How it works

claw-daw is built around a simple loop:

1) Define patterns (1–4 bars)
2) Place clips across sections
3) Export MIDI/WAV/MP3

Under the hood, MIDI is rendered through a GM SoundFont (via fluidsynth), then mastered/encoded via ffmpeg.

script-first arrangement 
# 4/4 timing (PPQ=480)
# 1 bar = 1920 ticks

new_project views_style_v1 74
add_track Drums 0
set_sampler 0 drums

new_pattern 0 d2 3840
add_note_pat 0 d2 36 0 180 112  # kick
add_note_pat 0 d2 38 1920 180 108 # snare

place_pattern 0 d2 15360 4  # hook
export_mp3 out/views_style_v1.mp3 preset=clean

Best practice: prompting a song

The fastest way to get a good result is to prompt like a producer brief: references + constraints + arrangement + palette + deliverables.

Copy/paste prompt template
Make a 2:45 instrumental with a late-night, introspective vibe
(think Drake “Views” era, but original—no melodies from the reference).

BPM: 74. Key: A minor-ish (dark/ambiguous). Time sig: 4/4.

Sections (bars):
- Intro: 8 bars (pad + soft keys only, lots of space)
- Hook: 8 bars (add drums + 808/sub + simple top motif)
- Verse: 16 bars (strip the top motif, keep drums sparse)
- Final hook/outro: 8 bars (bring motif back, then fade)

Sound palette: drums sampler + 808/sub + warm pad + EP/keys + light bell/pluck.
Drums: snare on 3, hats quiet, occasional ghost snare, simple kick pattern.
Mix targets: mono low end, gentle reverb, smooth highs.

Deliverables:
- tools/views_style_v1.txt
- out/views_style_v1.mp3 (preset=clean)
- plus project JSON + MIDI

FAQ (the stuff that usually breaks)

What’s a SoundFont?
It’s a .sf2 file that turns MIDI notes into actual instrument audio. claw-daw uses it via fluidsynth.

Why do I get “command not found” after install?
The install uses a user-level bin directory. Restart your terminal, or add $HOME/.local/bin (Linux) / $HOME/bin (some mac installs) to your PATH.

Is this a full DAW?
No—intentionally. It’s built for fast, reproducible sketches and automation.

How do I sanity-check my setup?
Run claw-daw doctor to verify fluidsynth, ffmpeg, and a SoundFont.

Get started

You’ll need fluidsynth, ffmpeg, and a SoundFont (.sf2). The installers below try to set this up for you.

Install (recommended)
curl -fsSL https://sdiaoune.github.io/claw-daw/install.sh | bash
Best on Debian/Ubuntu • attempts common distros • installs fluidsynth, ffmpeg, a GM SoundFont (when available), and claw-daw. 
curl -fsSL https://sdiaoune.github.io/claw-daw/install_mac.sh | bash
Requires Homebrew • installs fluidsynth, ffmpeg, and claw-daw. 
sudo apt-get update
sudo apt-get install -y fluidsynth ffmpeg fluid-soundfont-gm pipx
pipx reinstall "git+https://github.com/sdiaoune/claw-daw.git"
Use this if you prefer explicit package installs.
After install
claw-daw doctor
claw-daw --help
If claw-daw is not found, restart your terminal. Still not found? Try: export PATH="$HOME/.local/bin:$PATH" (Linux) or export PATH="$HOME/bin:$PATH" (some mac installs).
Update
Rerun the installer (safe) or use: pipx reinstall "git+https://github.com/sdiaoune/claw-daw.git"

Prefer automation? Run headless scripts from templates/ and export to out/.

What to expect (v1)

claw-daw v1 is intentionally minimal.

Great for: quick ideas, MIDI sketches, drum patterns, deterministic agent runs.
Not trying to be: a full replacement for Ableton/FL Studio.

If you want more realism, use a higher-quality SoundFont and sampler one-shots for drums.
